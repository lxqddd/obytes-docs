CI/CD 代表持续集成和持续交付/部署。它是一套实践，帮助开发者自动化构建、测试和分发应用程序到最终用户（如移动应用的应用程序商店）的过程。

为任何项目建立良好的 CI/CD 流程都至关重要，因为它可以帮助您节省时间和精力，同时确保向用户交付高质量的产品。它有助于避免常见错误和缺陷，缩短上市时间，并允许更快地向用户更新。

由于我们的大多数项目都托管在 GitHub 上，我们使用 [GitHub Actions](https://github.com/features/actions) 和 [Expo EAS](https://expo.dev/eas) 作为我们的 CI/CD 解决方案。启动器套件包含超过 10 个 GitHub Actions 工作流：

- `.github/workflows/lint-ts.yml`：在 PR 和主分支的新提交上，运行代码检查和格式化检查。
- `.github/workflows/test.yml`：在 PR 和主分支的新提交上，运行单元测试。
- `.github/workflows/type-check.yml`：在 PR 和主分支的新提交上，运行类型检查。
- `.github/workflows/compress-images.yml`：在主分支的新提交上，当添加图片时，创建一个包含压缩图片的拉取请求（PR）。
- `.github/workflows/expo-doctor.yml`：在 PR 和主分支的新提交上，检查依赖是否与 Expo SDK 版本保持一致。
- `.github/workflows/new-app-version.yml`：手动触发以更新应用版本并发布标签的工作流。

您可以在[此处](/ci-cd/workflows-references/)查看完整的工作流列表。

向商店推送新版本是我们团队的每周流程。为了简化这个过程，我们使用 GitHub Actions 和 Expo EAS 创建了一个简单的流程。您可以在[此处](/ci-cd/app-releasing-process/)查看完整指南。
