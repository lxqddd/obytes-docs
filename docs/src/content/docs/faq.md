本页面收集了关于 React Native / Expo Starter 的常见问题。

### 这个 Starter 适合我吗？

是的 😀

这个 Starter 旨在为广泛的 React Native 开发者提供帮助，从初学者到经验丰富的专业人士。以下是它可能适合您的原因：

1. **对于初学者：** 它提供了坚实的基础，包含最佳实践和常见解决方案，帮助您学习 React Native 开发的行业标准方法。

2. **对于经验丰富的开发者：** 它提供了结构良好、生产就绪的设置，可以节省您在项目初始化和配置方面的时间和精力。

3. **对于团队：** 它确保了项目和团队成员之间的一致性，使新开发者更容易加入并维护代码质量。

4. **对于探索者：** 即使您不喜欢使用 Starter 套件，这个项目也可以作为宝贵的参考。您可以探索代码库、文档和架构决策，以获得洞察，并可能为您的项目采用特定的解决方案。

5. **对于学习者：** 这个 Starter 包含了最新的库和模式，让您熟悉 React Native 生态系统中的当前最佳实践。

6. **对于 AI 辅助开发：** 这个 Starter 与 AI 编程工具配合良好。它提供了稳定的结构和最佳实践，可以指导 AI 生成的代码。这有助于确保 AI 辅助能够产生高质量、可维护的代码，并很好地融入您的项目。

请记住，您不必完全按照原样使用整个 Starter 套件。欢迎您根据项目需求挑选想法、配置或代码片段。无论您是从头构建新应用，还是希望改进现有的开发流程，这个 Starter 都可以提供宝贵的洞察和实用解决方案。

### 为什么选择 Expo 而不是 React Native CLI？

自从引入[持续原生生成 (CNG)](https://docs.expo.dev/workflow/continuous-native-generation/) 概念以来，我们一直使用 Expo 作为主要框架，我们对这个体验感到满意。

我认为这个问题不再有效，特别是自上次 React 大会以来，React Native 核心团队推荐新项目使用 Expo。

> "截至今天，React Native 唯一推荐的社区框架是 Expo。Expo 的团队从 React Native 早期就开始投资 React Native 生态系统，截至今天，我们相信 Expo 提供的开发者体验是最好的。" React Native 核心团队

还在犹豫？查看这篇[文章](https://reactnative.dev/blog/2024/06/25/use-a-framework-to-build-react-native-apps)或这个[视频](https://www.youtube.com/watch?v=lifGTznLBcw)，也许还有这个[视频](https://www.youtube.com/watch?v=ek_IdGC0G80)。

### 谁在维护这个 Starter？

这个 Starter 自 2021 年以来由 [Obytes 移动部落团队](https://www.obytes.com/team)维护。我们积极在内部项目中使用它，并很高兴与出色的 React Native / Expo 社区分享。这个项目是开源的，在 MIT 许可证下发布，确保了用户的灵活性。我们对这个 Starter 的承诺是持续的，只要它仍然是我们内部开发流程的核心部分，我们就会继续维护和改进它。我们欢迎社区的贡献和反馈，以帮助让这个 Starter 变得更好。

### 能包含 X 功能或库吗？

我们致力于保持 Starter 的简洁性。基于我们的经验，我们相信当前的设置为大多数项目提供了坚实的基础。

这个 Starter 是有主见的，反映了我们的最佳实践。如果您不同意我们的一些选择，那完全没问题。这个 Starter 被设计为易于定制，以满足您的特定需求。

我们经常收到通过 CLI 提供库和工具多种选择的请求。虽然这看起来很有吸引力，但它会显著增加复杂性和维护开销，这是我们试图避免的。
