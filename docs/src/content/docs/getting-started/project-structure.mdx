- `components/ui`：该文件夹包含所有的UI组件及主题配置。我们提供了基于 `obytes` 主题的基础组件，您可以在此添加自定义组件或主题配置。

- `components`：该文件夹存放应用的各类组件，主要用于 `app` 目录下。

- `lib`：该文件夹包含核心功能文件，例如 authentication、localization、storage 等，可在多个项目间共享。因此我们仅保留与业务无关的模块，这样更容易实现多项目的代码复用与 Starter 的升级。

- `app`：该文件夹用于存放应用的页面路由及其布局（如 stack/tab 等导航结构）。我们提供了基础导航结构，您可根据实际需求进行调整。

- `api`：该文件夹存放 API 相关功能。我们提供了基础 API client 和 provider，您只需为对应模块新增 query/mutation。可参考 `posts` 文件夹的实现方式。

- `translations`：该文件夹用于存放翻译资源文件。即使当前只支持单一语言，也建议使用此机制，便于未来扩展多语言，也有助于统一管理文案内容。

- `types`：该文件夹用于全局类型声明。

:::note
该结构较为主观，但我们认为这是构建 React Native 项目的最佳实践。当然你可以根据自己的需求做适当调整。
:::
