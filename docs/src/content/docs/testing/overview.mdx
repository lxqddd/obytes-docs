测试是任何软件开发过程中不可或缺的一部分。它有助于确保代码按预期工作，并避免引入错误到代码库中。由于我们重视测试的重要性，我们的启动模板让编写测试变得简单。我们为单元测试提供了 [Jest](https://jestjs.io/) 测试框架和 [React Native Testing Library](https://callstack.github.io/react-native-testing-library/)，以及对大多数库的模拟。我们还使用 [Maestro](https://maestro.mobile.dev/) 进行端到端测试。

## 测试方法

测试的重要性显而易见，但您应该测试什么呢？显然，对于前端应用（包括移动应用），没有明确的答案。我应该追求 100% 的代码覆盖率吗？我应该测试每个组件吗？

答案取决于情况。这取决于您的应用、团队和目标。我们认为测试是在编写测试的时间与从中获得的信心之间的权衡。您应该在这两者之间寻求良好的平衡。

追求 100% 的测试覆盖率并非总是好主意，也并非总是有意义。与仅渲染 UI 的视图和组件测试一样。相反，您应该关注以下内容：

- **业务逻辑**：测试包含业务逻辑的组件和功能工具。表单验证、数据操作和计算等。

- **复杂组件**：测试包含复杂逻辑的组件。例如，包含大量条件渲染的组件，或包含大量状态管理逻辑的组件。

专注于上述内容并为它们编写单元测试。这将帮助您从测试中获得最大收益，并协助您及早发现错误。

为了确保您的应用按预期运行，我们建议编写端到端测试，因为它们提供最高水平的信心，并帮助您像用户使用一样整体测试应用。
